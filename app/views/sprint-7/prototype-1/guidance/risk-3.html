High Risk

Some of the Council's software applications are no longer supported by the vendor

Why this is a risk

An easy and common entry point for ransomware and other malware is from rogue software or older software versions with out-of-date security features.

Not knowing what software you have and any lack of control over licenses increases this risk. Poor license management can also lead to employees downloading software that increases the risk of attack.

Without a software licensing inventory, costs can also increase and become unmanageable as well as the risk of penalties due to using unauthorised software.

Fines for unlicensed software can be incredibly high. If found guilty, an organisation can be fined a value of the court’s choosing and there is no limit to this amount.

What actions you should take

  • Produce a list of all your software licenses and create a repository to record them.
  • Ensure all users are made aware of software policies that make clear they cannot use unlicensed software.
  • Make sure you know your organisation’s license entitlement - which licenses you have and the legal right to use them.
  • Make sure you can prove you are entitled to them by documenting related information and contracts.
  • Lock down account controls.
  • Set alerts for whenever new software is installed so you can check it complies with licensing.
  • Set reminders for renewals and license reminders.
  • Track the use of software - any software that is no longer needed should be removed
  • Tracking the use of software can help reduce costs
